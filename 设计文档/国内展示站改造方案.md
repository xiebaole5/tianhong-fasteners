# 国内展示站改造方案

在原外贸独立站基础上，实现**国内部署的展示型网站**：产品可新增、不设价格、支持微信二维码，便于国内客户浏览与联系。

---

## 一、任务目标

- **保留**现有首页、产品页的视觉与结构，延续「外贸独立站」风格的网页展示效果。
- **支持**产品的新增与展示（无价格、仅展示名称/描述/图片等）。
- **增加**微信二维码展示位，方便国内客户扫码联系。
- **适配**国内部署环境（国内服务器/虚拟主机/对象存储 + 国内 CDN）。

---

## 二、输入依据

- 现有资源：`index.html`（首页）、`products.html`（产品页）、`config.php`、`images/products/` 产品图。
- 业务规则：展示型官网，不做在线下单与价格展示；联系以电话/微信/询盘为主。
- 约束：国内部署（无需 GitHub Pages 海外节点）；产品数据需可维护、可新增。

---

## 三、功能设计概览

| 功能点           | 说明 |
|------------------|------|
| 产品展示         | 列表/网格展示，支持分类筛选；无价格字段。 |
| 产品可新增       | 通过「数据文件」或「简易后台」新增产品（名称、描述、图片、分类）。 |
| 微信二维码       | 页脚/联系区/悬浮处展示二维码图，可配置图片路径。 |
| 国内部署         | 纯静态 + 可选 PHP，兼容虚拟主机/云服务器/OSS+COS。 |

---

## 四、技术方案

### 4.1 整体架构

```
[ 浏览器 ]
    ↓
[ 静态页 index / products ]  ← 从 data/products.json 读产品（JS 或 PHP 二选一）
    ↓
[ 产品数据 ]  data/products.json   （名称、描述、图片路径、分类，无价格）
[ 配置 ]      config.php 或 site-config.js  （站点名、微信二维码图片路径等）
```

- **产品数据**：统一放在 `data/products.json`，字段仅包含：`id`、`name`、`nameEn`、`description`、`image`、`category`、`sort`，**不包含价格**。
- **新增产品**：  
  - **方式 A（推荐）**：编辑 `data/products.json`，上传新图片到 `images/products/`，再上传/部署。  
  - **方式 B**：若使用 PHP 环境，可增加简易「管理页」通过表单写入 JSON 或数据库。
- **微信二维码**：在 `config.php` 或前端配置中增加「微信二维码图片路径」；页面在页脚、联系区块或固定悬浮位展示该图片。

### 4.2 产品数据结构（无价格）

```json
{
  "categories": [
    { "id": "hex-bolts", "name": "六角螺栓", "nameEn": "Hex Bolts" }
  ],
  "products": [
    {
      "id": "p1",
      "name": "六角头螺栓",
      "nameEn": "Hex Head Bolts",
      "description": "高强度工业用六角螺栓",
      "image": "images/products/hex-bolt-1.png",
      "category": "hex-bolts",
      "sort": 1
    }
  ]
}
```

- 不设 `price` 字段；若后续需要「询价」可保留「联系询价」按钮。

### 4.3 微信二维码展示位

- **位置建议**（可多选）：  
  1. 页脚「联系我们」旁：固定放一张微信二维码 + 文案「扫码加微信」。  
  2. 产品页 / 首页 CTA 区：在「发送询盘」「WhatsApp」旁增加「微信咨询」+ 二维码。  
  3. 右下角悬浮：与现有「WhatsApp 悬浮」并列，增加「微信」图标，点击展开二维码或跳转至带二维码的弹窗/区块。
- **配置**：在 `config.php` 中增加常量，例如 `WECHAT_QR_IMAGE = 'images/wechat-qr.png'`，前端用该路径渲染 `<img>`。

### 4.4 国内部署方式

| 方式 | 适用场景 | 说明 |
|------|----------|------|
| 虚拟主机（PHP） | 已有国内 PHP 空间 | 沿用现有 `config.php`，产品可用 PHP 读 JSON 或读 MySQL；上传 `data/products.json` 与图片即可。 |
| 纯静态 + 国内 CDN | 阿里云 OSS、腾讯云 COS、又拍云等 | 站点为纯 HTML/JS，产品由 JS 请求 `data/products.json` 渲染；微信二维码路径写相对路径或绝对 URL。 |
| 国内服务器（Nginx/Apache） | 自有云服务器 | 与虚拟主机类似，可 PHP 或纯静态；需备案。 |

- 若使用 **纯静态**：产品页通过 `fetch('data/products.json')` 动态渲染产品列表，新增产品只需更新 JSON 和图片并重新上传。  
- 若使用 **PHP**：可 `file_get_contents('data/products.json')` 或接入 MySQL，便于后续做简易管理后台。

## 五、页面与交互（执行结果）

### 5.1 首页（index.html）

- 保持现有结构；在「联系我们」或页脚增加**微信二维码区块**（图片 + 「扫码加微信」文案）。
- 若首页有 CTA（如「获取报价」），可在同一行增加「微信咨询」+ 二维码或图标。

### 5.2 产品页（products.html / products.php）

- **展示**：从 `data/products.json` 读取 `products` 与 `categories`，用现有卡片样式渲染；**不展示价格**；保留「询价/联系」按钮。
- **新增产品**：  
  - 使用方式 A：在 `data/products.json` 的 `products` 中追加一条，并上传对应图片到 `images/products/`。  
  - 使用方式 B：在管理页填写表单（名称、描述、分类、上传图片），提交后写入 JSON 或数据库。

### 5.3 页脚与悬浮

- 页脚：在联系信息旁增加微信二维码图 + 简短文案。
- 悬浮：在现有 WhatsApp 旁增加「微信」入口（图标或文字），点击可展示二维码或打开含二维码的弹窗。

### 5.4 验证要点（验证证据）

- 产品列表能正确从 `data/products.json` 展示，且**无价格显示**。
- 新增一条产品（改 JSON + 新图）后，刷新页面能看到新卡片。
- 微信二维码在至少一处（页脚/CTA/悬浮）正确显示且图片路径可配置。
- 在国内环境（虚拟主机或 OSS+CDN）访问正常，无依赖海外资源导致的不可用。

## 六、风险与限制

- **备案**：国内服务器或使用国内 CDN 的域名需完成 ICP 备案。
- **Google Fonts**：若部署在国内，部分环境访问 Google 字体可能较慢，可改为本地字体或国内 CDN 字体。
- **纯静态 CORS**：若 JSON 与页面不同域，需服务器配置 CORS 或改为同域存放 `data/products.json`。
- **管理方式**：若仅通过编辑 JSON 新增产品，需确保操作人员了解 JSON 格式，或提供简单说明/工具。
